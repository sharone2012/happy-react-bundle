👁 cfi_mill_full_profile view
Master read view. Joins mills + owners + soil in one query. COALESCE: uses confirmed soil type if available, province lookup as fallback. Soil confidence level derived automatically.
👁 cfi_tier1_targets view
Aggregated EFB + OPDC totals per TIER 1 owner. One row per company with SUM of annual EFB and OPDC tonnes across all their mills in the database.
👁 cfi_kcp_map view
KCP locations with owner join. Sorted by PKM output volume. Used to identify PKM supply sources near target mills.

Key Design Decisions
⚡
OPDC auto-calculated. est_opdc_t_yr is a GENERATED ALWAYS column = EFB × 0.152. You never enter OPDC volumes — the database derives them automatically from locked yield coefficient.
🔀
Soil fallback logic. Each mill stores a confirmed_soil_type. The full profile view uses COALESCE — confirmed value wins; if null, falls back to province lookup automatically.
🏗
Two schemas, different purposes. public = commercial pipeline (who to sell to, where, how much EFB). lab_analysis = scientific backbone (what's in the feedstock, what can go to BSF).
📐
Run order is mandatory. Migration 1 (public schema) must run before Migration 3 (lab schema). Migration 2 re-runs the soil lookup with cleaner formatting — drops and recreates that one table.
🔒
lab_analysis is a separate Supabase schema. Appears as its own schema tab. Query with prefix: SELECT * FROM lab_analysis.cfi_file_catalogue — not just the table name.
📊
Dynamic Model (xlsx) is separate. CFI_Dynamic_Model_v3.xlsx is the process calculator (PKSA kinetics, treatment optimizer, pathway calc). Not the same as the Supabase database — it's the operational engine.
