# CFI S1 — Processing Line (1-Pager) · Design Rules **Document:** CFI-S1-RULES-PL1P-001 Rev 01 **Date:** 30 March 2026 **Applies to:** EFB (S1C), OPDC (S1B), POS (S1A) --- ## 1. PURPOSE Every residue stream in S1 Pre-Processing gets an identical-structure **Processing Line (1-Pager)** — a single-page engineering summary showing the complete mechanical sequence from receiving to S2 handoff. The 1-Pager is the canonical visual reference for each stream. ## 2. GOLDEN RULE — 1:1 REPLICATION All three Processing Line 1-Pagers MUST be structurally identical. The only differences between EFB, OPDC, and POS are the data values, machine counts, stream-specific colours, and guardrail callouts. Layout, typography, spacing, card structure, and visual hierarchy are **locked**. ## 3. FRAME & SCALE | Rule | Value | |---|---| | Content scale | **60%** of original (shrunk 40%) | | Frame | Black outer border (3px solid #111) | | Padding field | White (#FFFFFF) background, 40px padding on all sides | | Outer background | Light grey (#E0E0E0) — matches CFI engineering drawing standard | | Content width | 1280px × 0.60 = 768px rendered, centred in frame | | Print target | A3 landscape or A4 landscape (scaled to fit) | ## 4. ANATOMY — TOP TO BOTTOM Every Processing Line 1-Pager has these sections in this exact order: ### 4.1 TOP BAR (stream-coloured) - Full-width bar, stream colour background - Left: stream identifier + drawing type + line code - Right: drawing number + revision + date + mill spec + throughput - Font: DM Mono 10px, white text **Stream colours (locked):** | Stream | Background | Accent | |---|---|---| | EFB S1C | #0B3A4A (dark teal) | #40C8C0 | | OPDC S1B | #2A1800 (dark amber) | #F5A623 | | POS S1A | #0B1422 (dark navy) | #40D7C5 | ### 4.2 LEFT PANEL (200px fixed width) Two sections, stacked: **a) Legend** — Icon swatches with labels: - Processing equipment (light fill, grey border) - Noise zone (red dashed border) — EFB/OPDC only - CLASS A gate (red solid border) — OPDC only - Gate checkpoint (amber border) - Flow direction arrow - Press water / drain (dashed line) **b) Machinery List** — Numbered tags for every machine: - Format: `[NUM] TAG-ID` + machine name + specs (elevation, kW, capacity) - Connector lines between machines: `▾ belt/conveyor ID · description` - Bottom: Line totals box (Installed kW, Throughput, Building dims, Floor slab, Power) ### 4.3 CONTENT AREA (fills remaining width) **a) Title block:** - Line: `{STREAM} line — {CODE}` (DM Sans 13px bold) - Subtitle: width, machine count, belt width, throughput, source, monthly volume - Warning box: MC in, lignin, C:N, target exit spec **b) Dimension line:** Width with tick marks and measurement (DM Sans 11px bold blue #185FA5) **c) Field Zone** — bordered container with badge: - Badge: `{CODE} · {STREAM} · {BUILDING DIMS}` (top-left corner overlay) - Contains ALL machine cards in sequence, top to bottom **d) Machine Cards** — the core repeating unit. Every machine gets one card: | Element | Spec | |---|---| | Image cell | 118×118px SVG schematic, DM Mono labels | | Data cell | Tag header (black #111 badge + tag ID), full machine name, field table | | Field table | 2-column pairs: Label/Value left, Label/Value right, separated by border | | Guardrail rows | Red bg for safety, amber bg for gates, blue bg for engineering notes | | Connector | Dashed bottom line showing next belt/conveyor + destination | **e) Special callouts (inline, between cards):** - Noise band: red dashed border, red text, hearing protection warning - CLASS A callout: red border, OPDC only, MC ≥ 40% locked **f) S2 Discharge box:** Stream-coloured rounded box at bottom, bold text **g) Height dimension:** Right-side vertical dimension line with measurement ## 5.
