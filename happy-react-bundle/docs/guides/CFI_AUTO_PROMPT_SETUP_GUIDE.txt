### **OPTION 1: SUPABASE DATABASE TRIGGER (RECOMMENDED)**

Store the master prompt in Supabase, then create a database function that loads it automatically.

#### **Step 1: Create table to store master prompts**

```sql
CREATE TABLE IF NOT EXISTS cfi_master_prompts (
  id SERIAL PRIMARY KEY,
  prompt_name TEXT UNIQUE NOT NULL,
  prompt_version TEXT NOT NULL,
  prompt_text TEXT NOT NULL,
  trigger_keywords TEXT[] NOT NULL,  -- e.g., ['add organism', 'new organism', 'upload organism']
  is_active BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Insert the master prompt
INSERT INTO cfi_master_prompts (prompt_name, prompt_version, prompt_text, trigger_keywords)
VALUES (
  'NEW_ORGANISM_ONBOARDING',
  'v2.0',
  '... [full CFI_MASTER_PROMPT_NEW_ORGANISM_v2.md text here] ...',
  ARRAY['add organism', 'new organism', 'upload organism', 'add new organism']
);
```

#### **Step 2: Claude checks this table when detecting trigger keywords**

```javascript
// In your app / Claude's logic:
const userMessage = "Add new organism: Thermobifida fusca";

// Check if trigger keyword present
const { data: prompts } = await supabase
  .from('cfi_master_prompts')
  .select('prompt_text')
  .eq('prompt_name', 'NEW_ORGANISM_ONBOARDING')
  .eq('is_active', true)
  .single();

if (prompts) {
  // Load prompt text and execute
  const masterPrompt = prompts.prompt_text;
  // Claude follows this prompt for the organism research
}
```

**PROS:**
- ✅ Prompt stored in database (version controlled)
- ✅ Easy to update without code changes
- ✅ Can enable/disable with `is_active` flag
- ✅ Multiple prompts for different triggers

**CONS:**
- ⚠️ Requires Supabase table creation
- ⚠️ Needs frontend or backend logic to check triggers

### **OPTION 3: GITHUB ACTION (MOST AUTOMATED)**

Create a GitHub Action that triggers when a new organism file is pushed to `/organisms/new/`.

#### **Step 1: Create GitHub workflow**

```yaml
# .github/workflows/organism-onboarding.yml
name: CFI Organism Onboarding

on:
  push:
    paths:
      - 'organisms/new/*.json'

jobs:
  onboard-organism:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Get organism file
        id: get-file
        run: |
          ORGANISM_FILE=$(ls organisms/new/*.json | head -1)
          echo "file=$ORGANISM_FILE" >> $GITHUB_OUTPUT
      
      - name: Run master prompt
        uses: anthropics/claude-action@v1
        with:
          prompt-file: ./CFI_MASTER_PROMPT_NEW_ORGANISM_v2.md
          input-file: ${{ steps.get-file.outputs.file }}
          output-file: ./organisms/processed/output.json
      
      - name: Upload to Supabase
        run: |
          # Your Supabase upload script here
          node scripts/upload-organism.js ./organisms/processed/output.json
```

**PROS:**
- ✅ Fully automated (push JSON → research → upload)
- ✅ GitHub-based version control
- ✅ Can run on schedule (e.g., nightly batch)

**CONS:**
- ⚠️ Most complex setup
- ⚠️ Requires GitHub Actions knowledge
- ⚠️ Need Claude API integration
