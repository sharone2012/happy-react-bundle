## 🎯 WHAT WE BUILT:

**User Flow:**
1. User clicks "Add Organism" button on any S3 page
2. Modal opens with search input
3. User types organism name (e.g., "thermobifda" — even misspelled!)
4. System searches ATCC/DSMZ/NRRL databases via Claude + web search
5. Shows top 5 matches with confidence scores (95%, 85%, 70%, etc.)
6. User selects closest match
7. Claude auto-executes master prompt research (13-professor panel, 99-power)
8. Shows preview: generic scores, 6 soil-specific scores, citations, warnings
9. User approves → Data loaded to Supabase with `is_approved = FALSE`
10. Sharon reviews and approves later

### **STEP 2: SET UP BACKEND API**

**Option A: Supabase Edge Functions (RECOMMENDED)**

Create three Supabase Edge Functions:

```bash
# In your Supabase project
supabase functions new organism-search
supabase functions new research-organism
supabase functions new insert-organism
```

Copy the code from `CFI_AddOrganism_API.js` into each function.

**Option B: Express Server**

If you already have a Node.js backend:

```javascript
// server.js
import express from 'express';
import { searchOrganism, researchOrganism, insertOrganism } from './CFI_AddOrganism_API.js';

const app = express();
app.use(express.json());

app.post('/api/organism-search', searchOrganism);
app.post('/api/research-organism', researchOrganism);
app.post('/api/insert-organism', insertOrganism);

app.listen(3000, () => console.log('API running on port 3000'));
```

## 🔒 SECURITY & SAFETY:

### **Approval Gate:**
- All organisms inserted with `is_approved = FALSE`
- Sharon must manually approve before production use
- Query to approve:
  ```sql
  UPDATE biological_library 
  SET recommended = TRUE 
  WHERE organism_name = 'Organism Name';

  UPDATE cfi_soil_organism_performance 
  SET is_approved = TRUE 
  WHERE organism_name = 'Organism Name';
  ```

### **Rate Limiting:**
Backend should implement rate limiting:
- 10 searches per minute per user
- 5 research workflows per hour per user
- Prevents abuse / excessive API costs

### **Input Validation:**
- Organism name: 3-100 characters, alphanumeric + spaces + hyphens
- SQL injection protected (using Supabase parameterized queries)
- XSS protected (React escapes all user input by default)

### **Error Handling:**
- Network errors: "Search failed. Please try again."
- Invalid organism: "No matches found. Try different spelling."
- Research errors: "Research workflow failed. Please try again."
- Database errors: "Database insert failed. Contact support."

## 🚨 TROUBLESHOOTING:

### **Problem: "No matches found"**
**Solution:** 
- Check Claude API key is valid
- Verify web search tool is enabled
- Try different organism name (scientific vs common)

### **Problem: "Research failed"**
**Solution:**
- Check master prompt is in Supabase (`cfi_master_prompts` table)
- Verify `is_active = TRUE` flag
- Check Claude API rate limits

### **Problem: "Database insert failed"**
**Solution:**
- Verify Supabase service role key
- Check table schemas match expected structure
- Look for constraint violations (duplicate organism name)

### **Problem: Modal doesn't open**
**Solution:**
- Check React component is imported correctly
- Verify no CSS conflicts (z-index issues)
- Check browser console for JavaScript errors

   ```bash
   cp CFI_AddOrganismButton.jsx src/components/

   ```

2. **Install dependencies:**
   ```bash
   npm install @anthropic-ai/sdk @supabase/supabase-js lucide-react
   ```

3. **Set environment variables:**
   ```bash
   # Production .env
   ANTHROPIC_API_KEY=sk-ant-...
   SUPABASE_URL=https://...
   SUPABASE_SERVICE_ROLE_KEY=...
   ```

4. **Deploy backend:**
   ```bash
   # Option A: Supabase Edge Functions
   supabase functions deploy organism-search
   supabase functions deploy research-organism
   supabase functions deploy insert-organism

   pm2 start server.js --name cfi-api
   ```

5. **Deploy frontend:**
   ```bash
   npm run build
   # Deploy to Vercel, Netlify, or your hosting provider
   ```
